Guideline

Token Rules

컴포넌트는 직접 색상·간격·그림자를 고정하지 않고, 가능한 한 디자인 토큰을 통해 표현합니다.

Token Usage
분류사용 기준대표 토큰
Color브랜드 색상은 accent 계열, 결과/상태는 semantic 계열만 사용합니다.--color-accent, --color-success, --color-danger
Spacing컴포넌트 내부 여백은 4px 스케일을 사용하고, 임의 px 값은 고정 크기 아이콘에만 제한합니다.--space-2, --space-4, --space-6
Radius입력·버튼·반복 항목은 md를 기본으로 두고, 큰 컨테이너만 lg 이상을 사용합니다.--radius-md, --radius-lg
Shadow일반 구분은 border를 우선하고, floating UI에만 그림자를 적용합니다.--shadow-xs, --shadow-lg
Motion상태 변화는 base, 오버레이·토스트 진입은 slow 범위 안에서 처리합니다.--transition-base, --transition-slow
 
.my-component {
  padding: var(--space-4);
  border: 1px solid var(--color-border);
  border-radius: var(--radius-md);
  background: var(--color-bg);
}

Guideline

State Rules

모든 조작 요소는 기본, hover, active, focus, disabled, loading, error 상태 중 필요한 상태를 일관된 방식으로 표현합니다.

Interaction State

Hover

색상 또는 배경만 살짝 변화시켜 탐색 가능한 요소임을 보여줍니다.

Active

버튼은 눌림 순간에 1px 이동 또는 약한 scale을 사용합니다.

Focus

키보드 포커스는 accent light ring으로 표시하며 hover와 구분합니다.

Disabled

opacity와 not-allowed cursor를 함께 사용하고 클릭 로직도 차단합니다.

Loading

중복 실행을 막고 cursor를 wait로 바꾸며 spinner를 함께 표시합니다.

Error

danger border와 보조 메시지를 함께 제공하고, 색상만으로 의미를 전달하지 않습니다.

<button class="btn btn-md btn-primary is-loading" aria-busy="true" aria-disabled="true">
  <span class="btn-spinner" aria-hidden="true"></span>
  저장 중
</button>

/* 2. 오류 표시 + 쉐이킹 + 스크롤 */
function showFieldError(input, msgEl, message) {
  input.classList.add('is-error');
  msgEl.textContent = message;
  msgEl.style.display = 'block';

  // 쉐이킹 — 이미 실행 중이면 재시작
  input.classList.remove('shake');
  requestAnimationFrame(() => input.classList.add('shake'));
  input.addEventListener('animationend', () => input.classList.remove('shake'), { once: true });
}

function clearFieldError(input, msgEl) {
  input.classList.remove('is-error');
  msgEl.style.display = 'none';
}

function validateAndSubmit(fields) {
  let firstError = null;

  fields.forEach(({ input, msgEl, validate, message }) => {
    if (!validate(input.value)) {
      showFieldError(input, msgEl, message);
      if (!firstError) firstError = input;
    } else {
      clearFieldError(input, msgEl);
    }
  });

  if (firstError) {
    // 첫 번째 오류 필드로 스크롤
    firstError.scrollIntoView({ behavior: 'smooth', block: 'center' });
    firstError.focus();
    return false;
  }
  return true;
}

function showToast(type, title, msg) {
  const toast = document.createElement('div');
  toast.className = 'toast toast-entering';
  toast.innerHTML = `
    <div class="toast-icon ${type}">...</div>
    <div class="toast-content">
      <div class="toast-title">${title}</div>
      <div class="toast-msg">${msg}</div>
    </div>
    <button class="toast-close" onclick="removeToast(this)" aria-label="토스트 닫기">✕</button>
  `;
  document.querySelector('.toast-container').appendChild(toast);
  setTimeout(() => removeToast(toast.querySelector('.toast-close')), 4000);
}

서비스 약관

제1조 (목적)
본 약관은 서비스 이용과 관련하여 필요한 사항을 규정합니다.

제2조 (정의)
서비스라 함은 회사가 제공하는 모든 서비스를 의미합니다. 이용자라 함은 본 약관에 동의하고 서비스를 이용하는 자를 말합니다.

제3조 (약관의 효력 및 변경)
본 약관은 서비스를 이용하고자 하는 모든 이용자에게 적용됩니다. 회사는 약관을 변경할 수 있으며, 변경된 약관은 공지 후 효력이 발생합니다.

제4조 (서비스의 제공)
회사는 서비스를 연중무휴, 1일 24시간 제공함을 원칙으로 합니다. 단, 시스템 정기점검 등 부득이한 사유가 있는 경우 서비스 제공을 일시 중단할 수 있습니다.

제5조 (이용자의 의무)
이용자는 관계 법령, 본 약관의 규정, 이용안내 및 서비스와 관련하여 공지한 주의사항, 회사가 통보하는 사항 등을 준수하여야 합니다.

제6조 (개인정보 보호)
회사는 이용자의 개인정보를 보호하기 위해 개인정보처리방침을 수립하고 이를 준수합니다.
